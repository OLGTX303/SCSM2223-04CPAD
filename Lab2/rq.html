Chapter 3 · Lab Exercise 2

Making It Dynamic
JavaScript & DOM Manipulation

Build a Kanban Task Board using pure JavaScript — no frameworks, no localStorage.

🎯 Exercise: My Task Board ⏱ Duration: 2–3 Hours 🟡 Difficulty: Beginner–Intermediate
2.1 Overview

You will build a Kanban-style Task Board in a single HTML file using vanilla JavaScript only — no libraries, no frameworks, and no localStorage (that comes in a later chapter).

The app lets users add tasks to one of three columns (To Do, In Progress, Done), edit and delete them, filter by priority, and inline-edit titles. All changes happen dynamically in the DOM without reloading the page.

2.2 Learning Outcomes
  • Select DOM elements using getElementById and querySelector
  • Create new elements with createElement, setAttribute, appendChild
  • Remove elements with element.remove()
  • Update element content safely with textContent
  • Add and remove CSS classes with classList.add / remove / toggle
  • Attach event listeners with addEventListener
  • Understand event delegation (one listener, many elements)
  • Implement simple in-page filtering without page reload
📖 DOM Methods Quick Reference
document.getElementById('id') Get one element by its id attribute
document.querySelector('.class') Get the first element matching a CSS selector
document.createElement('li') Create a brand-new HTML element
element.setAttribute('key', 'val') Set an attribute on an element (e.g. data-id)
element.textContent = 'text' Safely set or read the text inside an element
parent.appendChild(child) Insert a child element inside a parent
element.classList.add('cls') Add a CSS class to an element
element.classList.toggle('cls', bool) Add the class if bool is true, remove if false
element.remove() Remove an element from the DOM entirely
element.addEventListener('click', fn) Run a function when an event occurs
2.3 Scenario

Build a three-column Kanban board: To Do, In Progress, and Done. Users can add tasks via a modal pop-up, edit them, delete them, filter by priority, and double-click a title to rename it inline.

All work must be in three separate files: index.html, style.css, and taskmanager.js.

2.4 Tasks
Task 1

HTML Structure

  • Create a <header> with an app title and a task counter badge (updated by JS).
  • Add a priority filter <select> dropdown in the header.
  • Create three columns using <section id="todo">, <section id="inprogress">, <section id="done">.
  • Each column needs: a heading, a task list (<ul>), and an "Add Task" button.
  • Create a modal <div> (hidden by default) with: title input, description textarea, priority <select>, due date picker, Save and Cancel buttons.
Task 2

DOM Manipulation — Core CRUD Operations

Implement the following five functions in taskmanager.js:

  • createTaskCard(taskObj) — returns a <li> containing: title, description, priority badge, due date, Edit button, Delete button.
  • addTask(columnId, taskObj) — appends a new card to the right column and updates the task counter.
  • deleteTask(taskId) — adds a CSS fade-out animation class, then removes the card after the animation ends.
  • editTask(taskId) — opens the modal pre-filled with that task's existing data.
  • updateTask(taskId, updatedData) — updates the task object in the tasks array and refreshes the matching card's DOM content.
⚠️ You must NOT use innerHTML, innerText, or template literals to build cards.
All elements must be created with: createElement() · setAttribute() · classList.add() · appendChild() · textContent
Task 3

Event Handling

  • Event delegation: Attach a single click listener to each column's <ul> that handles both Edit and Delete button clicks using data-action and data-id attributes.
  • Inline editing: Double-clicking a task title replaces it with an <input>. Pressing Enter or moving focus away commits the change.
  • Priority filter: When a priority is chosen, hide non-matching cards using classList.toggle('is-hidden', condition) — not style.display.
  • Clear Done: A "Clear All" button removes every card from the Done column with a staggered fade-out (each card fades 100 ms after the previous).
💡 Event Delegation explained: Rather than adding a listener to every button, add ONE listener to the parent <ul>. When a button inside is clicked, the event bubbles up to the list. Check event.target.getAttribute('data-action') to see which action to run.
💡 Code Pattern: createElement vs innerHTML

The example below shows the correct approach (top) versus the approach you must NOT use (bottom). 

// ✅ CORRECT — use DOM API methods
const li = document.createElement('li');
li.setAttribute('data-id', task.id);
li.classList.add('task-card');

const titleSpan = document.createElement('span');
titleSpan.textContent = task.title;   // safe text — no HTML injection
li.appendChild(titleSpan);

// ❌ NOT ALLOWED — innerHTML builds cards
// li.innerHTML = `<span>${task.title}</span>`;  ← do not do this
💡 Code Pattern: Event Delegation
// ONE listener on the <ul> handles ALL button clicks inside it
todoList.addEventListener('click', function(event) {

  const action = event.target.getAttribute('data-action'); // 'edit' or 'delete'
  const idStr  = event.target.getAttribute('data-id');
  if (!action || !idStr) return; // click was on something else

  const taskId = parseInt(idStr, 10);

  if (action === 'delete') { deleteTask(taskId); }
  if (action === 'edit')   { editTask(taskId);   }
});
