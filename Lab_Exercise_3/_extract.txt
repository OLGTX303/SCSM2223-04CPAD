Chapter 4:  Lab Exercise 3 
Fetching Data — AJAX, JSON, Fetch API & jQuery 
 
Exercise Title WeatherNow —  Real-Time Data Dashboard with Fetch API & jQuery 
Duration 3 Hours (Laboratory Session) 
Topics Covered Fetch API, AJAX, JSON parsing, jQuery AJAX, CORS, Error handling, UI 
loading states 
Difficulty Intermediate —  Challenging 
 
4.1  Overview 
You will build a real-time Weather Dashboard that retrieves live data from two public APIs using both 
the native Fetch API and jQuery's $.ajax() method. You will implement proper loading states, error 
handling, data transformation, and a dynamic UI that updates without page reload. This exercise 
bridges the gap between static pages and truly data-driven web applications. 
 
4.2  Learning Outcomes 
• Understand the AJAX request/response cycle and the role of JSON 
• Make HTTP GET requests using the native Fetch API with async/await 
• Make HTTP requests using jQuery's $.ajax() and $.getJSON() methods 
• Parse and transform JSON responses into meaningful UI elements 
• Handle network errors, HTTP errors, and malformed responses gracefully 
• Implement skeleton loading screens and error state UI 
• Chain multiple dependent API calls to build enriched data views

4.4  Instructions 
Task 1 HTML & CSS —  Dashboard Shell with Skeleton Loaders 
 
1. Create a search bar at the top with a text input and Search button 
2. Create a 'current weather' card with placeholders for: city name, temperature, weather 
description, humidity, wind speed, and time 
3. Create a 7-day forecast row of 7 cards, each showing: day name, weather icon emoji, high/low 
temperature 
4. Implement skeleton loading animation: before data arrives, all card content areas must display a 
pulsing grey placeholder using CSS @keyframes animation 
@keyframes shimmer { 
  0%   { background-position: -200% 0; } 
  100% { background-position:  200% 0; } 
} 
.skeleton { 
  background: linear-gradient(90deg, #e0e0e0 25%, #f5f5f5 50%, #e0e0e0 75%); 
  background-size: 200% 100%; 
  animation: shimmer 1.5s infinite; 
  border-radius: 4px; 
}

Implement the following request chain using the Fetch API with async/await: 
5. On search: call the Geocoding API to resolve the city name into latitude and longitude
